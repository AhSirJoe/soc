02 / Core Concept

What is a System of Context?

Unlike a System of Record (what happened) or System of Engagement (how users interact), a System of Context captures why decisions were made, who was involved, what was known, and what should happen next.

System of Record

Stores what happened. Static data, historical transactions, state at a point in time.

System of Engagement

Manages how users interact. Workflows, notifications, collaboration interfaces.

System of Context

Captures why decisions were made, who was involved, reasoning, relationships, temporal state. The memory and intelligence layer for AI agents.

07 / Technical Deep Dive

Why Each Layer is Hard

Entity Resolution

"Same nurse in 5 systems" has no clean open-source solution. N² pairwise comparison problem at scale.

GraphRAG vs VectorRAG

GraphRAG outperforms on multi-hop reasoning (0% accuracy with 5+ entity chains in pure vector).

Bitemporal Memory

LLMs are stateless. Ghost facts, context overflow, temporal blindness require bitemporal architecture.

Real-Time Delivery

Batch context is wrong for operational agents. Need hybrid: batch cold layer + CDC streaming hot layer.

08 / Outlook

12-Month Technical Agenda

AI moves from decision support to decision execution. Three unsolved problems will define the next generation:

Unsolved #1

Temporal Memory at Scale

Bitemporal graph stores answering "what did the agent know at time T?" with sub-300ms latency. Zep, Cognee, Letta closest.

Unsolved #2

Contextual Entity Disambiguation

Same query from two users in different roles resolves to two different but both-correct outputs. Requires years of org data.

Unsolved #3

Agentic Governance by Intent

Policy engines evaluating what an agent is trying to do, not just what data it accesses. Lumia AI only pure-play.
